# Predicting the Validity of Set Data with a Self-supervised Masked Transformer

## Title

Predicting the Validity of Set Data with a Self-supervised Masked Transformer

## Description

This repository contains the official implementation and dataset preprocessing utilities for **SetMTr**, a self-supervised masked Transformer model designed for validity prediction of set-structured data.

SetMTr solves the problem of determining whether a given set is valid, coherent, or likely to originate from a real distribution. The model is trained using a masked reconstruction objective: a portion of a valid set is masked, the model reconstructs the missing elements, and the reconstruction quality serves as a validity score. Valid sets are reconstructed more accurately due to stronger underlying co-occurrence patterns.

**Repository Contents**:

- `setmtr/`: Core model implementation, data loaders, loss functions, metrics, utilities, and experiment configurations.
- `datasets/preprocess.py`: Converts text-format set data into pickle files used by the model.
- `datasets/`: Contains dataset format descriptions and preprocessing scripts.
- `datasets/original_data_process/`: Scripts and intermediate files for regenerating several benchmark datasets.

## Dataset Information

SetMTr is evaluated on 11 benchmark datasets spanning synthetic and real-world domains:

- **Chuancai and Yuecai**: Chinese recipe ingredient sets (Rong et al., IEEE Access, 2019).
- **Groceries**: Grocery transaction sets from the R `arules` package.
- **Inorganic compound**: Chemical element sets derived from CAS numbers.
- **Triangle**: Synthetic 2D point sets following triangle inequality rules.
- **SetMNIST**: 2D point sets derived from MNIST digits (10×10).
- **ModelNet10-sofa**: 3D point clouds sampled from ModelNet10 sofa category.
- **Metabolic reactions**: Reaction-metabolite sets from BiGG models (iAF1260b, iHN637, iIT341, iJO1366, etc.).

Additional auxiliary tasks include `market_basket`, `modelnet10_chair`, and various negative sampling variants.

**Data Format**:

- Text format (`datasets/txts/`): One set per line, elements separated by commas. Optional set name before colon.

  ```text
  Ba,I,H,O
  pineapple_pork:pork,pineapple,green_pepper,red_pepper
  563-63-3:Ag@[1],C@[2],H@[3],O@[2]
  ```

- Pickle format (`datasets/pkls/`): Generated by `preprocess.py`.

Some third-party raw datasets must be downloaded separately (see `original_data_process/`).

## Code Information

Key files and directories:

- `setmtr/main.py` — Training and evaluation entry point
- `setmtr/config.yaml` — Task selection, dataset paths, and hyperparameters
- `setmtr/models.py` — SetMTr model architecture
- `setmtr/modules.py` — Transformer encoder/decoder modules
- `setmtr/criteria.py` — Losses and evaluation metrics (AUC, F1, Hungarian matching, etc.)
- `setmtr/dataset.py` — PyTorch Dataset and DataLoader
- `datasets/preprocess.py` — Text-to-pickle conversion and negative sample generation
- Various scripts in `datasets/original_data_process/` for dataset regeneration

Built with PyTorch + PyTorch Lightning.

## Usage Instructions

1. **Clone the repository**

   ```bash
   git clone https://github.com/Tomas1861/setmtr-2026.git
   cd setmtr-2026
   ```

2. **Create and activate virtual environment**

   ```bash
   python -m venv .venv
   source .venv/bin/activate    # Windows: .venv\Scripts\activate
   ```

3. **Install dependencies**

   ```bash
   pip install -r requirements.txt
   ```

4. **Prepare datasets**

   - Place or generate text-format datasets in `datasets/txts/`

   - Run preprocessing:

     ```bash
     cd datasets
     python preprocess.py
     ```

5. **Configure and run**

   - Edit `setmtr/config.yaml` (set `task: groceries` or other tasks)

   - Train and evaluate:

     ```bash
     cd setmtr
     python main.py
     ```

## Methodology

All datasets are unified into a line-based text format. Sets with <2 elements are filtered. Negative samples are generated via element replacement or structural violation depending on the dataset type. SetMTr employs a permutation-invariant Transformer encoder-decoder with masked reconstruction training. Reconstruction error (via Hungarian matching for set alignment) serves as the validity signal.
